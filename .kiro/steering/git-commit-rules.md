# Git コミットルール

このプロジェクトでは、git操作において以下のルールを遵守します。

## 基本ルール

1. **個別ファイル追加**: `git add .` や `git add -A` を使用せず、ファイルを一つずつ追加する
2. **明確なコミットメッセージ**: 変更内容が分かるメッセージを記載
3. **論理的な単位でコミット**: 関連する変更をまとめてコミット

## 推奨される手順

```bash
# ファイルを一つずつ追加
git add path/to/file1.py
git add path/to/file2.py
git add path/to/file3.py

# コミット
git commit -m "明確な変更内容の説明"

# プッシュ
git push
```

## 禁止事項

- `git add .` の使用（全ファイル一括追加）
- `git add -A` の使用（全ファイル一括追加）
- `git add *` の使用（ワイルドカード一括追加）

## 例外

以下の場合のみ、複数ファイルの一括追加を許可：
- 新規プロジェクト初期化時の初回コミット
- 自動生成されたファイル群（例: CDK bootstrap）
- ユーザーが明示的に一括追加を指示した場合

## 理由

- 変更履歴の追跡が容易
- 意図しないファイルのコミットを防止
- レビューしやすいコミット単位の維持
